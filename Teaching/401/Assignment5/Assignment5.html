Idea: One common use of computers is for drawing images / pictures that can be saved and restored at a later time.  We can think of a primitive picture as a collection of polygons on the screen, and drawing a picture can consist of a user generating and manipulating these various polygons.  In this assignment you will complete a primitive polygon-drawing program so that a user can create and edit polygons on the screen in various ways.  The user will also be able to save the "picture" to a file and then restore it later if desired.

Assig5B program details:

You must complete this program to add some additional functionality to the original Assig5 program.  In particular, in addition to all of the functionality of Assig5, program Assig5B should also provide the following:

-      An additional Edit Menu item called "Push to Back".  This will push the selected MyPoly to the "back" of the picture so that other shapes can be drawn on top of it.

-      Several additional File Menu options:  "New", "Open", "Save" and "Save As".  "New" should clear the canvas and allow a new picture to be drawn.  "Open" should restore a properly formatted text file into the display.  "Save" should save the current scene into a text file, prompting for the file name if it does not already exist.  "Save As" should save the current scene into a new text file, whether or not it is currently stored in another file.  All of the File options should also keep track of the status of the scene – if it has been modified then the user should be asked to save it before doing anything that would destroy it ("New", "Open", "Exit").

-      An enhancement to MyPoly that will be utilized in Assig5B during the "Modify" mode.  During this mode the user needs to right click within a point circle of the MyPoly in order to remove the enclosed point.  However, the point circles are small and it may not be obvious to the user that the mouse is in fact within a point circle.  To help with this, when in Modify mode, if the mouse is moved to within one of the point circles within the selected MyPoly, that point circle (and no others) will be "filled" in rather than drawn in outline.  This change in appearance makes it clearer to the user that the mouse is inside the circle.  To do this you must add code to Assig5B.java and to MyPoly.  See more comments on this in file A5Bsnap.htm.

 

-      Carefully read over the API specifications of all classes and interfaces that you will be using.  In particular read over the Polygon, ArrayList, JPanel, Line2D, Line2D.Double, MouseEvent, and JFrame classes, and the MouseListener, MouseMotionListener and ActionListener interfaces.

Overriding contains to work in these cases is a requirement of this assignment.  Another example is handling the situation where all points in a previously drawn polygon are deleted – in this case the MyPoly should be deleted.  These cases are further explained in the program comments.
