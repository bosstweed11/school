You think the Rivers Casino is making too much money and want your cut of the action.  Thus, you have decided to design your own gambling computer program. Your program will allow a user to play the game of Over and Under (ok, so it is not taking the gambling community by storm, but it is simple and easy to implement!). In case you are not familiar with this staple of local festivals, Over and Under involves rolling two dice and trying to guess whether the result will be

 

  1. less than seven (under)
  2. seven exactly or
  3. greater than seven (over)

 

Before each roll of the dice, the player must make a choice of under, seven or over, and must make a bet (using compu-money, of course).  If a player's choice is correct, then he/she will receive the following winnings:

 

  1. if  (choice = = under)  or  (choice = = over), the user gets his/her bet back plus winnings equal to the bet.
  2. if  (choice = = seven) the user gets his/her bet back plus winnings equal to 4x the bet.

Thus, for example, if Joe Schmoe bets $5.00 on over and the dice come out as 9, Joe will get his $5.00 back plus another $5.00 in winnings. If Joe bets the same $5.00 on seven, and the dice come out as 7, Joe will get his $5.00 back plus $20.00 in winnings. Obviously, if Joe guesses incorrectly, the "house" keeps his hard-earned compu-dollars. Your assignment is to implement this Over and Under game in an object-oriented way in Java. Your implementation will have 3 major components:

 

  1. A class for the game player, called Player and stored in file Player.java. This will minimally include the names (first and last, as separate fields) of a player, his/her remaining money, the number of rounds he/she has played and the number of times he/she has won.  Naturally, it will also include mutator methods for manipulating the player in the necessary ways (remember a class contains data + operations and that the data should generally be private, so to mutate the data you must use methods).  Some examples of mutators might be "addMoney(double money)", "subtractMoney(double money)", "wonAGame()", "lostAGame()", etc.  You should also implement the toString() method to return the player's information as a nicely formatted String.
  2. A class for a die called Die and stored in Die.java.  This is a simple class that will store and allow the rolling of a single die.  For a given "roll" of a die the value should be randomly determined between 1 and 6.  The Die class should contain a Random object which will be used to generate the random values.  However, the Random object should be initialized only ONE time for each Die (ex: in the Die constructor).  For your game you will need two objects of the Die class, and a single roll will be the sum of the rolls of the two Die objects.
  3. A class (i.e. main program) for the game itself (or manager of the game, if you prefer), called Assig2 and stored in Assig2.java.  This program will read in a Player from a file (or create a new Player) and allow the Player to play the game in a user-friendly way (see details below).  In this program you will manipulate your Player and Die objects, as well as some other necessary variables.  Think carefully about the variables needed for your game.

The overall logic of your main program should be the following:

 

  1. Initially ask for and read in the player's first name from the keyboard.  Without loss of generality, assume a given player's name is Herb.  Now open a file equal to the player's name with the ".txt" extension.  For this example, the file name would be "Herb.txt".   If the file already exists, read the information from the file into a Player object.  If the file does not exist, prompt the user for the information and create the Player object in that way.  See the Java API for the File class to see how you can see if a file exists or not.  See the a2out.txt sample output to see an example of this process. [Note: You do not have to handle the situation in which you have two different players with the same first name, unless you want to do it for extra credit].
  2. Start the game.  A game consists of an arbitrary number of rounds.  Each round should proceed as follows:
    1. The program asks the user how much he/she wants to bet and inputs the value.  The amount must not exceed the cash on hand for the Player (your program must enforce this rule).
    2. The program asks the user what he/she wants to bet on (over, under or seven) and inputs the value.
    3. The program "rolls the dice" (via the two Die objects) and shows the user both die values (ex: 3 and 5) and their sum (ex: 8)
    4. The program automatically determines if the Player won or lost the round, informs the user and updates the money and rounds played / rounds won values in the Player object.
  3. The game should continue with rounds until one of two things occurs:
    1. The user elects to quit
    2. The Player is out of money
  4. At the end of the game, output the Player's current statistics to the display and then save the Player's information back to the file (same file name, <firstName>.txt).  Include in your output to the display the player's won/loss record for the last game, the player's overall (i.e. historic) won/loss record, and his/her net gain/loss in money during the game.  The idea of the file is that for a given Player the same file will be used from game to game, and will maintain the Player's information in between games.

See a2out.txt for a trace of how your program might look.  Your program does not have to match this exactly, but it should have the same functionality and approach to the game.
